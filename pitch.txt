PART 4: TECHNICAL FEATURES (For Judge Q&A)
================================================================================

COMPLETE TRAINING PIPELINE:
┌────────────────────────────────────────────────────────────────────────────┐
│                                                                            │
│   Standard Training    →    Curriculum Learning    →    Adversarial       │
│   (baseline)                (balanced learning)         Self-Play         │
│                                                         (target weakness) │
│                                                                            │
│   config.use_curriculum = True     True                 True              │
│   config.use_adversarial = True    False                True + LLM        │
│                                                                            │
└────────────────────────────────────────────────────────────────────────────┘

Our training used STANDARD mode. Results show WHY curriculum + adversarial needed.

CURRICULUM LEARNING (3 Phases):
1. Easy: Only obvious good/bad cases (Personal Loans)
2. Medium: Adds borderline cases (Vehicle Loans with LTV)
3. Hard: Adds all trap cases (Home Loans with RERA, tiered LTV)

ADVERSARIAL STRATEGIES (10 Total):
1. threshold_credit    - CIBIL 699 (1 point below)
2. threshold_foir      - FOIR 51% (1% above limit)
3. perfect_but_rera    - Dream profile, RERA=No
4. perfect_but_ltv_tier- >75L loan with LTV 76%
5. high_income_low_cibil - ₹8L income but CIBIL 695
6. employment_trap_home - Perfect home loan, 1 year employment
7. vehicle_ltv_trap    - Perfect vehicle, LTV 86%
8. docs_incomplete_good - Excellent profile, docs missing
9. coapplicant_trap    - Has co-applicant but FOIR>50%
10. borderline_multiple - Multiple metrics at exact thresholds

ADVERSARIAL TRACKER (Key Component):
- Tracks success/failure rate per strategy
- Identifies which strategy agent fails at most → get_weakness()
- Generates targeted batches: 70% weakness-focused, 30% random
- Provides summary of agent performance across all strategies

HOW ADVERSARIAL SELF-PLAY WORKS:
```
Round 1: Evaluate → Weakness: threshold_credit (80% failure)
         Generate 100 cases targeting threshold_credit
         Train → Improve on threshold_credit

Round 2: Evaluate → Weakness: perfect_but_rera (60% failure)
         Generate 100 cases targeting perfect_but_rera
         Train → Improve on RERA detection

Round 3: Evaluate → Weakness: vehicle_ltv_trap (50% failure)
         Generate 100 cases targeting vehicle_ltv_trap
         Train → Improve on LTV calculations
```
Result: Agent systematically improves on its weakest areas

TWO ADVERSARIAL MODES AVAILABLE:

1. RULE-BASED (Default - Free):
   - 10 predefined adversarial strategies
   - Deterministic and reproducible
   - Good for benchmarking
   
   ```python
   config.use_adversarial = True
   ```

2. LLM-POWERED (Advanced - Requires API Key):
   - Claude/GPT designs NOVEL trap cases
   - Adapts to agent's specific weaknesses  
   - More sophisticated and unpredictable
   - Like what the winning project used!
   
   ```python
   config.use_adversarial = True
   config.use_llm_adversarial = True
   config.llm_provider = "anthropic"  # or "openai"
   # Requires: ANTHROPIC_API_KEY or OPENAI_API_KEY env var
   ```

   LLM Adversarial Designer:
   - Receives agent's weakness analysis
   - Generates loan applications designed to TRICK the agent
   - Creates cases that "look approvable but have one hidden flaw"
   - Similar to how Kube SRE Gym used Claude for incident design

REWARD STRUCTURE (Asymmetric):
- Correct decision: +10
- Approve bad loan: -15 (3× worse than rejecting good loan)
- Reject good loan: -5
- Approve non-RERA: -20 (worst case - regulatory risk)

WHY ASYMMETRIC REWARDS?
"In banking, NPA losses (principal + interest + recovery costs) far exceed
the opportunity cost of a declined good customer. Our rewards mirror this."
